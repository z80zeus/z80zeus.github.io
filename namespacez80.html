Вариант "преобразования" IP-адреса в строку из... строки. На самом деле никакого преобразованяи не производится. Функция возвращает строку, переданную параметром.

Template Parameters
Tstd::string
Parameters
ipКонстантная ссылка на строку. Функция не анализирует содержимое переданной строки и возвращает её как есть.
Returns
Для T = std::string функция возвращает объект типа std::string. Для T других типов данный вариант функции не скомпилируется.

template<typename T >
std::enable_if_t< std::conjunction_v < z80::has_const_iterator<T>, z80::has_no_c_str<T> >, std::string > z80::ip2string ( const T &  ip)

Преобразование представления IP-адреса из содержимового контейнера - в строку.

Template Parameters
TТип - контейнер.
Parameters
ipконстантная ссылка на объект типа T.
Returns
Для T типа контейнер - возвращает строку, составленную из содержимого контейнера, разделённого точками. Для типа T не предоставляющего необходимый интерфейс контейнера - данный вариант функции не скомпилируется.

Для работы функции требуется наличие константного итератора - это проверяется метафункцией z80::has_const_iterator<T>.
Однако под данный критерий попадает и std::string, который имеет интерфейс практически идентичный контейнерам, но std::string должен обрабатываться другим вариантом этой функции.
Для исключения применения данного варианта функции, std::string отбрасывается метафункцией z80::has_no_c_str<T>, которая не должна обнаружить наличие метода c_str у типа T: в std::string этот метод есть, у контейнеров - нет.
Результаты работы этих двух метафункций: z80::has_const_iterator<T> и z80::has_no_c_str<T> - собираются по "И" метафункцией std::conjunction.
